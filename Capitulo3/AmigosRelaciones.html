Amigos y Relaciones

Los enlaces regulares ( a href) simplemente apuntan a otra página. Las relaciones de enlace son una forma de explicar por qué
estás apuntando a otra página. Terminan la frase “Estoy señalando esta otra página porque:”
es una hoja de estilo que contiene reglas CSS que su navegador debería aplicar a este documento.
es un feed que contiene el mismo contenido que esta página, pero en un formato de suscripción estándar.
es una traducción de esta página a otro idioma.
es el mismo contenido que esta página, pero en formato PDF.
es el próximo capítulo de un libro en línea del cual esta página también forma parte.

Y así. HTML5 divide las relaciones de enlace en dos categorías :
Se pueden crear dos categorías de enlaces utilizando el elemento de enlace. Los enlaces a recursos externos son enlaces a recursos que
se utilizarán para aumentar el documento actual, y los enlaces de hipervínculo son enlaces a otros documentos.
De los ejemplos anteriores, solo el primero ( rel="stylesheet") es un enlace a un recurso externo. El resto son hipervínculos a
otros documentos. Es posible que desee seguir esos enlaces, o puede que no, pero no son necesarios para ver la página actual.

REL = HOJA DE ESTILO
link rel="stylesheet" href="style-original.css" type="text/css"
Esta es la relación de enlace más utilizada en el mundo (literalmente). link rel="stylesheet" es para apuntar a las reglas CSS que se
almacenan en un archivo separado. Una pequeña optimización que puede hacer en HTML5 es eliminar el typeatributo. Solo hay un
lenguaje de hoja de estilo para la web, CSS , por lo que ese es el valor predeterminado para el typeatributo. Esto funciona en todos
los navegadores. (Supongo que alguien podría inventar un nuevo lenguaje de hoja de estilo algún día, pero si eso sucede, simplemente typevuelva a agregar el atributo).
link rel="stylesheet" href="style-original.css"

REL = ALTERNATIVO
link rel="alternate"
type="application/atom+xml"
title="My Weblog feed"
href="/feed/"
Esta relación de enlace también es bastante común. link rel="alternate", combinado con el tipo de medio RSS o Atom en el typeatributo,
habilita algo llamado "detección automática de fuentes". Permite a los lectores de fuentes sindicadas (como Google Reader )
descubrir que un sitio tiene una fuente de noticias con los artículos más recientes. Algunos navegadores también admiten la detección
automática de feeds al mostrar un icono especial junto a la URL . (A diferencia de rel="stylesheet", el typeatributo importa aquí.

La rel="alternate"relación de enlace siempre ha sido un extraño híbrido de casos de uso, incluso en HTML 4 . En HTML5 , su
definición se ha aclarado y ampliado para describir con mayor precisión el contenido web existente. Como acaba de ver, el uso
rel="alternate"junto con type=application/atom+xmlindica un feed Atom para la página actual.
HTML5 también pone fin a una confusión de larga data sobre cómo vincular a traducciones de documentos. HTML 4 dice que use el
langatributo junto con rel="alternate"para especificar el idioma del documento vinculado, pero esto es incorrecto.

OTRAS RELACIONES DE ENLACES EN HTML5

rel="author"se utiliza para enlazar a información sobre el autor de la página. Puede ser una mailto:dirección, aunque no tiene por
qué serlo. Simplemente podría vincular a un formulario de contacto o a una página "sobre el autor".

rel="icon" es la segunda relación de enlace más popular , después de rel="stylesheet". Por lo general, se encuentra junto con shortcut, así:
link rel="shortcut icon" href="/favicon.ico"
Todos los principales navegadores admiten este uso para asociar un pequeño icono con la página. Por lo general, se muestra en la barra
de ubicación del navegador junto a la URL, en la pestaña del navegador o en ambas.

rel="license" fue inventado por la comunidad de microformatos . “Indica que el documento al que se hace referencia proporciona los
términos de la licencia de derechos de autor bajo los cuales se proporciona el documento actual”.

rel="nofollow" "indica que el enlace no está respaldado por el autor original o el editor de la página, o que el enlace al documento
al que se hace referencia se incluyó principalmente debido a una relación comercial entre las personas afiliadas a las dos páginas".

rel="noreferrer" "indica que no se filtrará información de referencia al seguir el enlace". WebKit es compatible con
rel="noreferrer" , por lo que funciona en Google Chome y Safari

rel="prefetch" "indica que es probable que obtener y almacenar en caché de manera preventiva el recurso especificado sea
beneficioso, ya que es muy probable que el usuario requiera este recurso". Los motores de búsqueda a veces agregan a la página de
resultados de búsqueda si sienten que el resultado principal es mucho más popular que cualquier otro.

rel="buscar" "indica que el documento al que se hace referencia proporciona una interfaz específica para buscar el documento y sus
recursos relacionados". Específicamente, si desea rel="search"hacer algo útil, debe apuntar a un documento de OpenSearch que
describe cómo un navegador podría construir una URL para buscar una palabra clave determinada en el sitio actual.

rel="tag" "indica que la etiqueta que representa el documento al que se hace referencia se aplica al documento actual". Technorati
inventó el marcado de "etiquetas" (palabras clave de categoría) con el relatributo para ayudarlos a categorizar las publicaciones de blog.
