Section

El sectionelemento representa un documento genérico o una sección de aplicación. Una sección, en este contexto, es una agrupación temática de contenido, generalmente con un encabezado. Ejemplos de secciones serían los capítulos, las páginas con pestañas en un cuadro de diálogo con pestañas o las secciones numeradas de una tesis.

El navelemento representa una sección de una página que enlaza con otras páginas o partes dentro de la página: una sección con
enlaces de navegación. No todos los grupos de enlaces en una página deben estar en un navelemento; solo las secciones que constan de
bloques de navegación principales son apropiadas para el navelemento. En particular, es común que los pies de página tengan una breve
lista de enlaces a páginas comunes de un sitio, como los términos de servicio, la página de inicio y una página de derechos de
autor. El footerelemento solo es suficiente para tales casos, sin un navelemento.

El articleelemento representa un componente de una página que consta de una composición independiente en un documento, página,
aplicación o sitio y que está destinado a ser distribuible o reutilizable de forma independiente, por ejemplo, en sindicación. Esto
podría ser una publicación en un foro, un artículo de revista o periódico, una entrada de registro web, un comentario enviado por un
usuario, un widget o dispositivo interactivo, o cualquier otro elemento de contenido independiente.

El asideelemento representa una sección de una página que consta de contenido que está tangencialmente relacionado con el contenido alrededor del asideelemento, y que podría considerarse separado de ese contenido. Estas secciones a menudo se representan como barras laterales en la tipografía impresa.

El hgroupelemento representa el encabezado de una sección. El elemento se utiliza para agrupar un conjunto de h1– h6elementos cuando el encabezado tiene varios niveles, como subtítulos, títulos alternativos o eslóganes.

El headerelemento representa un grupo de ayudas introductorias o de navegación. Por lo general, un headerelemento debe contener el encabezado de la sección (un h1elemento h6o un hgroupelemento), pero esto no es obligatorio. El headerelemento también se puede usar para envolver la tabla de contenido de una sección, un formulario de búsqueda o cualquier logotipo relevante.

El footerelemento representa un pie de página para su ancestro más cercano que secciona el contenido o secciona el elemento raíz. Un pie de página generalmente contiene información sobre su sección, como quién lo escribió, enlaces a documentos relacionados, datos de derechos de autor y similares. Los pies de página no necesariamente tienen que aparecer al final de una sección, aunque normalmente lo hacen.
