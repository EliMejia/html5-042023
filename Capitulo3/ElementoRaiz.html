EL ELEMENTO RAIZ

Una página HTML es una serie de elementos anidados. Toda la estructura de la página es como un árbol. Algunos elementos son
“hermanos”, como dos ramas que se extienden desde el mismo tronco de árbol. Algunos elementos pueden ser "hijos" de otros elementos,
como una rama más pequeña que se extiende desde una rama más grande. (También funciona a la inversa; un elemento que contiene otros
elementos se denomina nodo "principal" de sus elementos secundarios inmediatos y "ancestro" de sus nietos). Los elementos que no
tienen elementos secundarios se denominan nodos "hoja". El elemento más externo, que es el antepasado de todos los demás elementos de la página, se denomina "elemento raíz". El elemento raíz de una página HTML siempre es "html"

Pero los elementos en HTML5 siempre están en este espacio de nombres , por lo que ya no necesita declararlo explícitamente. Su
página HTML5 funcionará exactamente igual en todos los navegadores, ya sea que este atributo esté presente o no. Dejar caer el xmlnsatributo nos deja con este elemento raíz:

html lang="en" xml:lang="en"

Los dos atributos aquí, langy xml:lang, definen el idioma de esta página HTML . ( ensignifica "inglés". ¿Por qué dos atributos para
la misma cosa? De nuevo, esto es un vestigio de XHTML . Solo el langatributo tiene algún efecto en HTML5 . Puede mantener el
xml:langatributo si lo desea, pero si lo hace, debe asegurarse de que contenga el mismo valor que el langatributo .
Para facilitar la migración hacia y desde XHTML, los autores pueden especificar un atributo sin espacio de nombres sin prefijo y con el
nombre local literal "xml:lang" en elementos HTML en documentos HTML

Eso nos deja con este elemento raíz:
html lang="en"
"en" significa ingles, se debe buscar el adecuado, para español es "es-419", por lo que quedaria html lang="es-419"
