CAMPOS DE ENFOQUE AUTOMÁTICO

ESTABLECER EL ENFOQUE LO ANTES POSIBLE Muchas páginas web esperan hasta window.onloadque se disparan para establecer el foco. Pero el window.onload evento no se activa hasta
que se hayan cargado todas las imágenes. Si su página tiene muchas imágenes, una secuencia de comandos tan ingenua podría volver a
enfocar el campo después de que el usuario haya comenzado a interactuar con otra parte de su página. Esta es la razón por la que
los usuarios avanzados odian los scripts de enfoque automático.

El ejemplo de la sección anterior colocó la secuencia de comandos de enfoque automático inmediatamente después del campo de formulario
al que hace referencia. Esta es la solución óptima, pero puede ofender su sensibilidad si coloca un bloque de código JavaScript en
el medio de su página. (O, de manera más mundana, es posible que sus sistemas de back-end no sean tan flexibles). Si no puede
insertar un script en el medio de su página, debe establecer el enfoque durante un evento personalizado como el de jQuery en
$(document).ready()lugar de window.onload.

EJEMPLO

Para resumir: 1.-Establecer el enfoque correctamente es importante.

2.-Si es posible, deje que el navegador lo haga configurando el autofocusatributo en el campo de formulario en el que desea tener el foco.

3.-Si codifica un respaldo para navegadores más antiguos, detecte la compatibilidad con el autofocusatributo para asegurarse de que su
respaldo solo se ejecute en navegadores más antiguos.

4.-Establezca el enfoque lo antes posible. Inserte el script de enfoque en su marcado inmediatamente después del campo de formulario.
Si eso le ofende, use una biblioteca de JavaScript que admita eventos personalizados y active un evento personalizado inmediatamente después
del marcado del campo de formulario. Si eso no es posible, use algo como el evento de jQuery $(document).ready().

5.-Bajo ninguna circunstancia debe esperar hasta window.onloadestablecer el enfoque.
