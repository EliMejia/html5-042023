Al igual que con el marcado de personas , debe establecer los atributos itemscopey en el elemento más externo. itemtype. En este caso,
el elemento más externo es un article elemento. El itemtype atributo declara el vocabulario de microdatos que está utilizando
(en este caso, http://data-vocabulary.org/Organization), y el itemscopeatributo declara que todas las propiedades que establece en
los elementos secundarios se relacionan con este vocabulario.

De acuerdo con el modelo de datos de microdatos de HTML5 , los h1 elementos no necesitan ningún procesamiento especial. El valor de la
propiedad de microdatos es simplemente el contenido de texto del h1 elemento.
Si desea enumerar más de un número de teléfono, tal vez uno para clientes de los Estados Unidos y otro para clientes internacionales,
puede hacerlo. Cualquier propiedad de microdatos se puede repetir. Solo asegúrese de que cada número de teléfono esté en su propio
elemento HTML , separado de cualquier etiqueta que pueda darle.

En este ejemplo, la información de geolocalización no es así. No hay texto visible que dé la latitud y la longitud exactas de la
Organización. De hecho, el ejemplo de organization.html (sin microdatos) no tiene ninguna información de geolocalización. Tiene un
enlace a Google Maps, pero incluso la URL de ese enlace no contiene coordenadas de latitud y longitud.
Para manejar casos extremos como este, HTML5 proporciona una forma de anotar datos invisibles . Esta técnica solo debe usarse como
último recurso. Si hay una forma de mostrar o representar los datos que le interesan, debe hacerlo. Los datos invisibles que solo
las máquinas pueden leer tienden a "obsoletarse" rápidamente. Es decir, alguien vendrá más tarde y actualizará el texto visible
pero se olvidará de actualizar los datos invisibles. Esto sucede más a menudo de lo que piensas, y también te sucederá a ti. La información de geolocalización se define en su propio vocabulario, como la dirección de una Persona u Organización . Por lo tanto, este span elemento necesita tres atributos:

1.- itemprop="geo"dice que este elemento representa la geopropiedad de la Organización circundante

2.- itemtype="http://data-vocabulary.org/Geo"dice a qué vocabulario de microdatos se ajustan las propiedades de este elemento

3.- itemscopedice que este elemento es el elemento que encierra un elemento de microdatos con su propio vocabulario (dado en el
itemtypeatributo). Todas las propiedades dentro de este elemento son propiedades de http://data-vocabulary.org/Geo, no del entorno http://data-vocabulary.org/Organization.

De acuerdo con el modelo de datos de microdatos de HTML5 , el meta elemento tiene un procesamiento especial. El valor de la propiedad
de los microdatos es el contentatributo. Dado que este atributo nunca se muestra visiblemente, tenemos la configuración perfecta
para cantidades ilimitadas de datos invisibles.
