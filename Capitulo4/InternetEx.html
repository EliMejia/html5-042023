¿QUÉ PASA CON IE?

Las versiones de Internet Explorer anteriores a la 9.0 no admiten la API de lienzo. Sin embargo, esas versiones
anteriores de Internet Explorer sí son compatibles con una tecnología patentada de Microsoft llamada VML , que
puede hacer muchas de las mismas cosas que el canvaselemento. Y así excanvas.jsnació.

Explorercanvas ( excanvas.js) es una biblioteca JavaScript de código abierto con licencia de Apache que
implementa la API de lienzo en Internet Explorer. Para usarlo, incluya el siguiente script elemento en la parte superior de su página.

Los bits [if lt IE 9] son comentarios condicionales . Internet Explorer los interpreta como una declaración: "si
el navegador actual es una versión de Internet Explorer anterior a (pero sin incluir) la versión 9, ejecute
este bloque". Todos los demás navegadores tratarán el bloque completo como un comentario HTML .
El resultado neto es que Internet Explorer 7 y 8 descargarán el script y lo ejecutarán, pero otros navegadores
ignorarán el script por completo (no lo descargarán, no lo ejecutarán, nada).

Hay algunas limitaciones:

**Los degradados solo pueden ser lineales. No se admiten degradados radiales .

**Los patrones deben repetirse en ambas direcciones.

**Las regiones de recorte no son compatibles.

**La escala no uniforme no escala correctamente los trazos.

**Es lento. Esto no debería sorprender a nadie, ya que, para empezar, el analizador de JavaScript de Internet
Explorer es más lento que otros navegadores. Una vez que comience a dibujar formas complejas a través de una
biblioteca de JavaScript que traduce los comandos a una tecnología completamente diferente, las cosas se
atascarán. No notará la degradación del rendimiento en ejemplos simples como dibujar algunas líneas y transformar
una imagen, pero lo verá de inmediato una vez que comience a hacer animación basada en lienzo y otras cosas locas.

Hay una advertencia más sobre el uso de excanvas.js, y es un problema con el que me encontré al crear los
ejemplos de este capítulo. ExplorerCanvas inicializa automáticamente su propia interfaz de lienzo falso cada vez
que incluye el excanvas.jsscript en su página HTML . Pero eso no significa que Internet Explorer esté listo
para usarlo de inmediato. En ciertas situaciones, puede encontrarse con una condición de carrera en la que la
interfaz de lienzo falso es casi , pero no del todo, lista para usar.
